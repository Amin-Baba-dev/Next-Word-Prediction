<span style="font-size: 24px;">strategy</span> 

First, turn the raw text into numbers. Every unique character (letters, spaces, punctuation) gets a unique integer ID. The entire corpus is then converted into a long sequence of these integers. Instead of trying to feed the whole text into the model at once, we chop this integer stream into fixed‑length blocks (say, 100 characters per block). We deliberately discard any leftover characters that don’t form a complete block, keeping things simple and batch‑compatible.

Second, turn each block into a learning exercise. For every block, we create two versions: the original block minus its last character becomes the input, and the same block minus its first character becomes the target. This forces the model to predict the next character at every single position inside the block. We then randomly shuffle these input‑target pairs, group them into small batches, and set up a background loader to keep the GPU fed. That’s it – no overlapping windows, no complicated state management, just clean, contiguous chunks that teach the network to guess what letter comes next.
